# QA vs QC vs Testing

## Part A: Comparison Chart

| Aspect | QA (Quality Assurance) | QC (Quality Control) | Testing |
|--------|------------------------|----------------------|---------|
| Definition | Процес, насочен към **предотвратяване** на дефекти в процесите, които създават продукта. | Процес, насочен към **откриване и коригиране** на дефекти в готовия продукт. | Дейност за **идентифициране на грешки или несъответствия** в продукта чрез изпълнение на тестове. |
| Focus | Процесите и системите, които водят до качествен продукт. | Крайните продукти и техните характеристики. | Конкретни функционалности или компоненти на продукта. |
| Approach | Проактивен (предотвратяване на дефекти). | Реактивен (откриване на дефекти). | Изпълнение на тестови случаи за проверка на функционалностите. |
| Timing | По време на разработката / целия жизнен цикъл на продукта. | След разработката, преди или по време на доставка. | След изграждане на функционалности или компоненти (unit, integration, system testing). |
| Responsibility | Целият екип за качество и процеси. | QC екип или тестери, понякога QA специалисти. | Тестери или QA специалисти изпълняват тестовете. |
| Example Activity | - Разработка на стандарти и процеси<br>- Провеждане на процесни одити<br>- Обучение на екипа за добри практики | - Преглед на кода<br>- Инспекция на документация<br>- Проверка на готовите функционалности срещу спецификацията | - Unit тестове<br>- Integration тестове<br>- UI/UX тестове<br>- Regression тестове |

## Part B: Real-World Examples

### QA Examples
1. Създаване на стандарти за писане на код и кодови ревюта в екипа.
2. Провеждане на процесен одит за следване на Agile/Scrum практики.
3. Организиране на обучение за безопасност на данните и добри практики при разработка.

### QC Examples
1. Преглед на техническата документация за грешки или пропуски.
2. Визуална проверка на интерфейса на уебсайт спрямо дизайн макети.
3. Инспекция на финален продукт за работещи функционалности преди release.

### Testing Examples
1. Провеждане на Unit тестове за отделни функции.
2. Провеждане на Integration тестове за взаимодействие между модули.
3. Regression тестове след нови промени, за да се уверят, че старите функционалности не са нарушени.

## Part C: Scenario Classification

1. Executing test cases for the login feature → **Testing**
2. Establishing coding standards for the development team → **QA**
3. Conducting code reviews before merging pull requests → **QC**
4. Running automated regression tests → **Testing**
5. Creating a checklist for deployment process → **QA**
6. Reporting a bug found during test execution → **QC**
7. Implementing a peer review process → **QA**
8. Validating that requirements are testable → **QA**
