## Exercise 1: Test Level Identification (30 minutes)

### Objective
Practice categorizing test scenarios into appropriate test levels.

---

| # | Test Scenario | Level | Justification |
|---|---------------|-------|---------------|
| 1 | Verify `calculateTotal()` function returns correct sum | **Unit** | Тества се отделна функция от програмата |
| 2 | User can log in and see products page | **System** | Проверява се цяла потребителска функционалност|
| 3 | Shopping cart updates when product is added via API | **Integration** | Проверява взаимодействието между frontend и backend чрез API. |
| 4 | Password encryption function works correctly | **Unit** | Тестваме конкретна логика за криптиране|
| 5 | Complete purchase flow from login to confirmation | **System** | Пълно end-to-end тестване|
| 6 | Database stores user credentials correctly | **Integration** | Проверява се връзката между приложението и базата данни. |
| 7 | Frontend displays data from backend API | **Integration** | Фокусът е върху интеграцията между два компонента – UI и API. |
| 8 | New user can complete first purchase without help | **Acceptance** | Проверява се цялостен юзер сценарий |
| 9 | Sorting function arranges products alphabetically | **Unit** |Проверка на конкретна функция за сортиране. |
| 10 | Payment gateway integrates with checkout system | **Integration** | Тества се взаимодействието на два компонента |
