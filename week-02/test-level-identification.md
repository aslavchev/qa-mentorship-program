## Exercise 1: Test Level Identification (30 minutes)

### Objective
Practice categorizing test scenarios into appropriate test levels.

---

| # | Test Scenario | Level | Justification |
|---|---------------|-------|---------------|
| 1 | Verify `calculateTotal()` function returns correct sum | **Unit** | Testing separate functions of the program|
| 2 | User can log in and see products page | **System** | Checking the whole functionality|
| 3 | Shopping cart updates when product is added via API | **Integration** | Checking interaction between frontend and backend via API|
| 4 | Password encryption function works correctly | **Unit** | Testing the encryption logic|
| 5 | Complete purchase flow from login to confirmation | **System** | Full end-to-end testing|
| 6 | Database stores user credentials correctly | **Integration** | Checking interaction between the app and the database|
| 7 | Frontend displays data from backend API | **Integration** | UI and API components and how they interact with each other|
| 8 | New user can complete first purchase without help | **Acceptance** | User full scenario |
| 9 | Sorting function arranges products alphabetically | **Unit** | Checking separate function for sorting |
| 10 | Payment gateway integrates with checkout system | **Integration** | Two components and how they work together |
