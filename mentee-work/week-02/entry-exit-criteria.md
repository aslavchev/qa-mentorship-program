## Exercise 3: Entry/Exit Criteria 

### Objective
Define clear entry and exit criteria for each test level.

### Instructions

Complete the following tables for SauceDemo testing:

### Unit Testing (if you had access to code)

**Entry Criteria:**
- [ ] Code written and compiles
- [ ] Unit test framework set up
- [ ] Tools set up for testing 
- [ ] Clear and easily readable code

**Exit Criteria:**
- [ ] All unit tests pass
- [ ] Code coverage > 80%
- [ ] Regression testing complete to ensure no additional bugs arise
- [ ] Most important parts of the code have 100% test coverage

### Integration Testing

**Entry Criteria:**
- [ ] Unit testing complete
- [ ] Modules ready for integration
- [ ] Integration test environment available
- [ ] All defects from unit testing are closed and ready
- [ ] Test data is ready

**Exit Criteria:**
- [ ] All integration tests pass
- [ ] API contracts validated
- [ ] Modules are stable and working well together
- [ ] All bugs found are documented and cleared

### System Testing

**Entry Criteria:**
- [ ] Integration testing complete (>90% pass)
- [ ] QA test environment stable
- [ ] All features development complete
- [ ] Test cases reviewed and approved
- [ ] Documentation for the requirements is ready

**Exit Criteria:**
- [ ] All system tests executed (>95%)
- [ ] Pass rate > 90%
- [ ] No critical/high defects open
- [ ] Every bug found is documented 
- [ ] Confirmation testing is done to ensure the program beahves as intended

### Acceptance Testing

**Entry Criteria:**
- [ ] System testing complete
- [ ] All P0/P1 defects resolved
- [ ] Staging environment ready
- [ ] UAT participants available
- [ ] No unresolved defects on system level

**Exit Criteria:**
- [ ] All UAT scenarios passed
- [ ] Business sign-off received
- [ ] No showstopper issues
- [ ] Product does what it should and how it should based on client demands
- [ ] Documentation is complete
